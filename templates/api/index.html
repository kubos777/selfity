Lista de endpoints disponibles

Para comenzar a utilizar los siguientes endpoints, es necesario realizar la creación de un usuario en el sistema:

Objeto JSON prueba:

{ "telephone": nuevo_numero, "password ": contraseña(más de 8 caracteeres), }

Objeto JSON prueba:

{ "telephone": tu_número_celular }

Endpoint que recibe un número celular válido (10 dígitos), se comprobará que el mismo no tenga una sesión activa, de no tener una sesión activa, se debe enviará un SMS con un código aleatorio de 6 dígitos.

Objeto JSON prueba:

{ "telephone": tu_número_celular }

Endpoint que recibe un número celular válido (10 dígitos) y su respectivo código de 6 digitos, al ser un código válido y estar relacionado con el número celular, regresará un Bearer token para la autenticación de futuras peticiones.

Objeto JSON prueba:

{ "telephone": tu_número_celular, "code": codigo_recibido }

Endpoint para verificar si la sesión sigue activa.

Objeto JSON prueba:

{ "telephone": tu_número_celular }

Endpoint que recibe una imagen en base64, además se reciben los siguientes campos:
  • MIME Type
  • Hashtag(Utilizado como llave primaria)
  • Coordenadas (latitud, longitud)
  • Archivo en base 64


Objeto JSON prueba:

{ "MIMEType": formato_imagen, "hashtag": hashtag, "coords": coordenadas_lat_lng, "file": imagen_base_64 }

Endpoint que recibe el ID(hashtag) del archivo(imagen) y regresará una miniatura de la misma (no la original), la fecha de creación y el hashtag. Petición GET: https:://kubos.mx/api/fifth/data-thumbnail/hashtag

Endpoint que regresa una lista de todas las imagenes almacenadas. Petición GET: https://kubos.mx/api/sixth/images/list

Endpoint que regresa una lista de hashtags. Petición GET: https://kubos.mx/api/seventh/hastags/list
